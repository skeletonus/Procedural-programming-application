# Графический учебный тренажёр по процедурному программированию

**Процедурный тренажер** — это интерактивное приложение для изучения основ процедурного программирования через различные форматы: карточки для запоминания, практические задачи с написанием кода, а также теорию и тесты для закрепления знаний.

## Цель проекта

Разработать удобный, автономный и интуитивно понятный учебный тренажер, который:

– Позволяет создавать и редактировать карточки с вопросами и ответами по программированию;

– Предоставляет практические задания по написанию кода с автоматической проверкой;

– Содержит структурированную теорию и интерактивные тесты;

– Не требует установки и работает без подключения к интернету.

## Основные возможности

– **Карточки** — создавайте каталоги карточек, добавляйте, редактируйте и удаляйте записи для запоминания терминов и концепций;

– **Написание кода** — решайте практические задачи, пишите код прямо в приложении, запускайте тесты и сохраняйте решения;

– **Теория и тесты** — изучайте материалы по модулям и темам, проходите тесты с автоматической проверкой и отслеживанием прогресса;

– **Локальное хранение** — все данные (карточки, задачи, теория) хранятся в локальных папках в форматах JSON и TXT;

– **Полная автономность** — работает без интернета, установки и дополнительных зависимостей;

– **Интерфейс на русском языке**.

## Используемые технологии

– **Язык программирования** – Python 3.13+;

– **Графический интерфейс** – Tkinter;

– **Обработка данных** – JSON, обычные текстовые файлы;

– **Сборка в .exe (опционально)** – PyInstaller.

```bash
# Клонируйте репозиторий (если есть) или скопируйте файлы проекта
# Запустите start.cmd или через консоль
```

Все данные (карточки, задачи, теория) автоматически создаются в папках:
- `cards_dir/` — каталоги карточек
- `code_dir/` — задачи для написания кода
- `theory_tests_dir/` — теоретические материалы и тесты

## Тестирование

Проект включает функциональные тесты для проверки всех основных модулей:

– Система прогресса (задачи, тесты);

– Работа с карточками (создание, редактирование, удаление);

– Загрузка и отображение теории;

– Проверка тестовых заданий;

– Блок написания кода с запуском и проверкой.

### Запуск тестов:

```bash
pytest test.py
```

Все тесты проходят успешно — подтверждена корректность работы основных функций приложения.

## Структура проекта

```
Процедурный-тренажер/
├── data                            # Папка с данными
│   ├── cards_dir/                  # Папка с карточками
│   │   └── card1.json              # Данные карточки
│   ├── code_dir/                   # Папка с задачами
│   │   └── [task_folder]/          # Папка задачи
│   │       ├── task.json           # Условие задачи
│   │       └── solution.txt        # Решение задачи
│   └── theory_tests_dir/           # Папка с теорией и тестами
│       └── [module]/               # Модуль
│           └── [topic]/            # Тема
│               ├── theory.txt      # Текст теории
│               └── test.json       # Тестовые вопросы
├── src                             # Папка с кодом
│   └── application                 # Папка с кодом
│       ├── __init__.py             # обозначает директорию как пакет
│       ├── cards.py                # Реализация карточек
│       ├── coding.py               # Реализация окна для написания кода
│       ├── main.py                 # Запуск
│       ├── menu.py                 # Главное меню
│       ├── mk_scrollable_frame.py  # Создание фрейма со скроллбаром
│       ├── test.py                 # Тестирование
│       └── theory.py               # Теория
├── .gitignore                      # Игнорирует файлы при копировании
├── .python-version                 # Версия python
├── LICENSE                         # Лицензия
├── README.md                       # О проекте
├── pyproject.toml                  # Общая информация о проекте
├── start.cmd                       # Файл запуска
└── uv.lock                         # Фиксирует пакеты
```
